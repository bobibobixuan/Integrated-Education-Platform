# Python Adventure Game — 打包部署说明

## 快速启动（开发/测试环境）

直接双击 `start.bat`，脚本会自动检测 Python 并启动服务。

```
start.bat              # 默认端口 80，完整模式（学生端+教师端）
start.bat 8000         # 指定端口 8000
start.bat --admin      # 仅教师端模式
start.bat 8000 --admin # 端口 8000 + 仅教师端
```

- 如果端口 80 需要管理员权限或被占用，服务会自动回退到 8000
- 数据库文件自动创建在 `data/app.db`（与 `start.bat` 同级目录）
- 可通过环境变量 `GAME_PORT` 预设端口号

## 打包为独立 EXE

### 1. 构建

双击 `build.bat`，或手动执行：

```bash
pip install pyinstaller
pip install -r requirements.txt
pyinstaller --clean pyinstaller.spec
```

构建输出在 `dist/game/` 目录。

### 2. 部署

将整个 `dist/game/` 文件夹复制到目标机器，然后运行 `game.exe`。

```
dist/game/
├── game.exe          # 主程序（双击启动）
├── data/             # 数据库目录（首次运行自动创建）
├── _internal/        # Python 运行时和依赖
└── dev/              # 前端静态资源
```

**启动参数**（命令行运行）：
```
game.exe                 # 默认端口 80，完整模式
game.exe --port 8000     # 指定端口
game.exe --admin         # 仅教师端
```

### 3. 首次运行

1. 双击 `game.exe`（或 `start.bat`）
2. 浏览器访问 `http://localhost`（端口 80）或 `http://localhost:8000`（回退端口）
3. 数据库自动初始化在 `data/app.db`

## 数据库说明

- 位置：`data/app.db`（SQLite，与启动程序同级目录）
- 首次运行自动创建表结构和执行迁移
- 备份：直接复制 `data/` 整个文件夹即可
- 包含：用户数据、题库、答题记录、PVP 对战数据、排行榜

## 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `GAME_PORT` | 服务端口 | `80` |
| `GAME_STATIC_MODE` | `student`=完整模式 `admin`=仅教师端 | `student` |
| `PYTHONPATH` | Python 模块搜索路径 | 自动设为启动目录 |

## 教师账号

首次启动后需要手动创建教师账号：
1. 注册一个普通学生账号
2. 在 `data/app.db` 中执行：
   ```sql
   UPDATE users SET role = 'admin' WHERE username = '你的用户名';
   ```

## 局域网访问

启动后控制台会打印局域网 IP，同网络内的设备可通过该 IP 访问。例如：
```
http://192.168.1.100/
http://192.168.1.100/admin/
```

## 生产部署建议

1. 修改 `JWT_SECRET`：通过环境变量或 `.env` 文件设置随机密钥（至少 64 字符）
2. 建议使用 Nginx 反向代理（HTTPS + 静态资源缓存）
3. 定期备份 `data/` 目录
4. Windows 防火墙需放行对应端口
