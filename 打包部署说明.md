# Python Adventure Game — 打包部署说明

## 快速启动（开发/测试环境）

直接双击 `start.bat`，脚本会自动检测 `backend\.venv` 中的 Python，并启动完整服务。

```
start.bat              # 默认端口 80，学生端 + 教师端一起启动
```

- 当前 `start.bat` 固定以 `80` 端口启动，并调用 `python -m backend.app.main --port 80`
- 脚本要求本地已存在 `backend\.venv`；如果缺失，请先执行 `uv sync --directory backend`
- 数据库文件自动创建在 `data/app.db`
- 教师端访问路径为 `/app/admin`

## 打包为独立 EXE

### 1. 构建

优先使用项目虚拟环境中的 Python 执行 PyInstaller。当前仓库主要使用：

```bash
backend\.venv\Scripts\python.exe -m PyInstaller pyinstaller_onefile.spec --clean
```

如需目录式输出，可改用：

```bash
backend\.venv\Scripts\python.exe -m PyInstaller pyinstaller.spec --clean
```

### 2. 部署

当前仓库支持两种发布方式：

1. 单文件发布：使用 `pyinstaller_onefile.spec`，产出单个 `exe`
2. 目录式发布：使用 `pyinstaller.spec`，复制整个输出目录

无论哪种方式，前端构建资源都会一起打包，目标机器不需要额外安装 Python。

公开仓库中不应提交 `dist/` 下的构建产物，应只保留源码、脚本和文档。

### 3. 首次运行

1. 运行打包后的 `exe`，或在开发环境双击 `start.bat`
2. 浏览器访问 `http://localhost/`
3. 管理端访问 `http://localhost/app/admin`
3. 数据库自动初始化在 `data/app.db`

## 数据库说明

- 位置：`data/app.db`（SQLite，与启动程序同级目录）
- 首次运行自动创建表结构
- 备份：直接复制 `data/` 整个文件夹即可
- 包含：用户数据、题库、答题记录、PVP 对战数据、排行榜

## 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `GAME_PORT` | 服务端口 | `80` |
| `GAME_DEV_MODE` | 是否启用开发模式 | `false` |
| `PYTHONPATH` | Python 模块搜索路径 | 由启动脚本设置 |

## 教师账号

首次启动时，系统会自动补种默认管理员账号：

- 用户名：`admin`
- 密码：`admin123`

首次登录后需要立刻修改密码。

## 局域网访问

启动后控制台会打印局域网 IP，同网络内的设备可通过该 IP 访问。例如：
```
http://192.168.1.100/
http://192.168.1.100/app/admin
```

## 生产部署建议

1. 修改 `JWT_SECRET`：通过环境变量或 `.env` 文件设置随机密钥（至少 64 字符）
2. 首次启动后尽快修改默认管理员密码
3. 定期备份 `data/` 目录
4. Windows 防火墙需放行对应端口
