# 1.0.7

## 安全与防作弊

- 修复答题 Session 清理逻辑，过期 Session 现在会正确标记为 `expired`，避免重复开局时状态残留。
- 修复状态同步接口过度信任前端进度的问题，改为服务端根据真实答题记录重建关卡进度。
- 强制改密门禁并入统一鉴权链，`force_password_change` 用户现在无法绕过改密直接使用其他接口。
- 调整答题防作弊的最短用时处理：低于 2 秒的作答不再直接拒收，改为按最小 2 秒记账，避免学生端本地通关但后端完全无记录。
- 修复最终一题提交后的 Session 完成判定，确保 Session 会进入 `completed`，连击、关卡战力和在线时长能正确落库并被教师端看到。
- 修复 2 秒保底计时导致的战力误判，速度分和连击分恢复正常计算。

## 学生端

- 修复开局异步建 Session 的竞态，答题不再因为 `play_session_id` 还没回来而静默丢记录。
- 补齐轻量 toast 提示，替换原来会退化成 `alert()` 的交互。
- 重做学生端首页信息层级与主界面入口布局，提升首次进入时的可理解性。
- 学生端登录页现在会持久化显示登录状态；刷新后会恢复账号显示，并按注册开关隐藏或展示注册入口。
- 重做学生端排行榜 UI，增加公开访问提示、清晰表头和个人名次区。
- 排行榜改为匿名可见：未登录学生也能直接查看战力榜和在线时长榜，登录后才额外展示个人名次。
- 将原“周活跃榜”统一调整为“在线时长榜”，后端按有效答题记录的累计 `time_spent` 汇总，学生端结算与榜单文案同步更新。
- 关卡结算补齐战力明细面板，显示本关战力、总战力、在线时长和四维分项，不再额外弹独立结算窗。

## 教师端

- 重做教师端整体视觉主题、仪表盘头部和登录卡样式，并把学生列表操作改为事件委托。
- 新增学生端自助注册开关：后端默认保留能力但可关闭，教师端仪表盘可控制状态。
- 新增学生批量导入接口和教师端导入入口，支持按 `username/nickname/password` 批量创建学生账号。
- 新增题目 `title` 字段并接入题目管理、错题统计和导入规范，教师端不再只能靠题干长文本定位题目。
- 扩充教师端仪表盘数据分析，新增薄弱单元、高表现学生、待跟进学生、推进率与练习强度等细分指标。
- 重做教师端登录门禁：后台页面默认以未认证状态渲染，登录前直接隐藏侧栏和全部管理内容。

## 排行榜与后端稳定性

- 修复选项 JSON 解析兼容性问题，避免题目配置在部分场景下崩溃。
- 修复排行榜按昵称判定“我的排名”的错误，改为按 `user_id` 匹配，避免同名用户串榜。
- 修复在线时长榜接口响应模型不匹配导致的加载失败。
- 修复学生正常通关后排行榜战力仍为 0 的问题：关卡完成时立即回写 `UserStats.power_score`。
- 移除战力计算服务里的隐式 `db.commit()` 副作用，避免读取型接口提前提交事务。

## 启动与部署

- 调整 `start.bat` 启动提示：默认端口仍为 `80`，并在启动时明确打印游戏页和教师端地址，便于区分当前访问地址。
- 新增 `start.bat` 启动脚本，提升 Windows 本地启动稳定性。
