2.0.0 版本更新：全栈重组

## 后端重构 (backend/)
- 创建 backend/ 项目骨架，使用 uv 管理依赖与虚拟环境
- 配置 FastAPI + SQLAlchemy 2.0 + pydantic-settings + structlog 核心依赖栈
- 新增 app/config.py（Settings 配置类，支持 .env 读入与 PyInstaller 冻结模式路径）
- 新增 app/database.py（SQLAlchemy 引擎、SessionLocal、get_db/session_scope 双模式会话管理、SQLite WAL pragma）
- 新增 app/exceptions.py（AppError 业务异常体系：AuthError / ForbiddenError / NotFoundError / ConflictError / RoomFullError / BattleExpiredError）
- 迁移 14 个 ORM 模型到 backend/app/models/，补全双向 relationship（User 从零关系增至 9 个反向引用）
- PvpRoomMember.is_ready 从 Integer(0/1) 改为 Boolean，所有 DateTime 列统一 timezone=True
- 所有 UniqueConstraint 通过 ORM __table_args__ 声明（5 个模型 6 个约束）
- 新增 Alembic 迁移系统 (alembic/)，独立 CLI 入口 (python -m app migrate)，带文件锁防并发
- 迁移 Service 层：auth_service / question_service / scoring_service / pvp_service / rate_limiter
- 迁移 Router 层：auth / units / questions / records / scores / achievements / leaderboard / pvp / admin
- 双前缀路由：/api/v1/（新版）+ /api（旧版兼容，include_in_schema=False）
- 新增 scoring 单元测试（calc_stars / calc_combo / is_extreme_pass / calc_power_bonus）共 14 个
- 新增 auth 集成测试（注册/登录/重复用户名/密码错误）共 3 个
- 后端全部 18 个测试通过

## WebSocket 模块
- 拆离 ws/ 独立包：manager.py（连接管理）/ router.py（端点注册）/ auth_cache.py（鉴权缓存 10s TTL）
- 新增 pytest-asyncio + httpx-ws WebSocket 心跳超时断连测试

## 结构化日志与异常处理
- 新增 app/logging_config.py（structlog 控制台+JSON 双输出）
- 新增 app/exceptions.py 统一异常注册与处理
- 新增 app/main.py（lifespan 管理 / 双环境构建 / StaticFiles 挂载 frontend/dist/）

## 前端重构 (frontend/)
- 创建 frontend/ 项目骨架：Vite 8 + Vue 3 + TypeScript + Pinia + Vue Router 4
- Vue Router 使用 createWebHistory('/app/')，所有视图组件懒加载
- 新增 auth API 封装、Pinia 认证 store、7 条路由及 7 个页面组件（Login / Game / PvPLobby / PvPBattle / Leaderboard / Achievements / Admin）
- 新增 Vitest + Vue Test Utils + happy-dom 单元测试框架，编写 LoginView 和 GameView 组件测试
- 新增 Playwright E2E 测试骨架（登录页面加载验证）
- npm 脚本新增 test / test:watch / test:e2e 三条命令

## 构建与部署
- 更新 start.bat 指向新 backend/ 目录，使用 uv run 启动 FastAPI 服务
- 更新 build.py 适配 frontend/dist/ Vue SPA 构建产出与 /app/ 基础路径
- 更新 pyinstaller.spec 入口改为 backend/app/main.py，补齐所有 backend.app.* 隐藏导入
- 更新 .gitignore 适配新项目布局（忽略 frontend/dist/、backend/.venv/、*.db 等）
